# 30-Second Demo Script

Use this script when recording the README demo GIF or short launch video.

## Goal

Show a real LanceDB inspection workflow in under 30 seconds. The viewer should understand three things before the clip ends:

- this is read-only
- it works on local LanceDB data
- it replaces throwaway inspection scripts

## Recording Checklist

- Start with the app closed.
- Keep the sample dataset folder ready.
- Use a window size large enough to show the sidebar and schema panel.
- Avoid terminal footage except for a single optional title card.

## Suggested Shot List

### 0s to 4s

Title card or caption:

`Inspect LanceDB locally without writing scripts`

### 4s to 9s

Open LanceDB Viewer and show the empty state briefly.

Caption:

`Read-only desktop viewer for LanceDB`

### 9s to 15s

Click the folder button and select the sample data directory.

Caption:

`Open your LanceDB root folder`

### 15s to 22s

Click a table and land on the schema tab.

Highlight:

- row count
- vector dimension
- field names

Caption:

`Check schema and vector metadata in seconds`

### 22s to 28s

Switch to the data tab and show the first rows.

Caption:

`Preview rows without a notebook or script`

### 28s to 30s

End frame:

`Download: github.com/wufeng1740/lanceDB-viewer`

## Export Targets

- GIF for README and social posts
- 45 to 90 second MP4 for GitHub release, X, LinkedIn, Reddit, and dev.to embeds

## Copy Options For Social Clips

- `Stop writing throwaway scripts just to inspect LanceDB.`
- `A safer way to browse LanceDB locally.`
- `Useful for RAG debugging when you need to inspect tables fast.`
